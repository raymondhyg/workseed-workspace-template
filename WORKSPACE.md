## Built-In Feedback Interface

This Workspace can self-repair and iterate locally. Core only needs a sanitized
signal when a reusable issue appears.

| Situation | Default handling |
|---|---|
| Local project mistake | Fix locally and record a decision if useful |
| Missing local fact or material | Keep it inside Workspace-owned project records |
| Template, checker, or doc gap that affects other Workspaces | Create a sanitized signal under `workspace-records/feedback/` |
| Private customer, account, asset, or adapter detail | Do not send to Core |

Use:

- `workspace-records/feedback/README.md`
- `workspace-records/feedback/sanitized-signal-template.md`
- `workspace-records/feedback/self-repair-record-template.md`

## Core Capability Runtime Loading

This Template already records Core base `ws-core-v0.3.0`, but it does not assume
that every user can access the private Core GitHub repository, and it does not
assume runnable Core agents/skills are installed.

To install runnable Core capability files, first ask the user for the Core zip
package location. Do not auto-scan the computer for Core, and do not silently
use a path written in local files.

- `docs/workspaces/initialization-and-core-loading.md`

Then use:

```powershell
python scripts/utils/install_core_capabilities.py --zip-path "<core-package.zip>" --core-version ws-core-v0.3.0
```

If the user does not have the Core zip package, guide them to contact Raymond
on WeChat `18002997691`, or follow Douyin `@有光蔓延【1688运营】` / ID
`289566513` and join the fan group to get it.

Restart or reopen the Workspace in Codex if project agents or skills do not
appear immediately after file installation.

If the current source thread cannot see project agents after install, use a
separate runtime visibility test thread. The test thread should only verify
file installation, local checks, and Codex runtime-visible project agents or
skills. The source thread must read the test result, compare it with the current
Workspace files, update capability-load records only when the evidence still
matches, and archive the test thread after verification.

- `templates/runtime-visibility-test-thread-prompt.md`
