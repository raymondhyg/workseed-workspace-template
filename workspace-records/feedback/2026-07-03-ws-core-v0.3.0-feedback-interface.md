# 2026-07-03 ws-core-v0.3.0 Built-In Feedback Interface

Generated by: `scripts/utils/create_workspace_deployment.py`

Workspace: WorkSeed Workspace Template
Core release: `ws-core-v0.3.0`
State: Applied as Template default

## Purpose

This interface lets the Workspace self-repair and iterate locally while keeping
Core feedback sanitized and reusable.

## Flow

1. Diagnose locally.
2. Decide whether the issue is Workspace-local or reusable.
3. Record self-repair evidence if the Workspace fixed it.
4. Create a sanitized Core signal only when the issue may improve Core or the
   Workspace Template for other users.

## Classification

| Classification | Use when | Destination |
|---|---|---|
| `Workspace self-repair` | Local Codex can fix it with Workspace-owned facts or records | `workspace-records/feedback/` and/or `workspace-records/decisions/` |
| `Workspace record only` | Useful locally, not reusable by Core | `workspace-records/feedback/` |
| `Core sanitized signal` | Reusable Template/Core gap with private facts removed | sanitized issue, PR, or Core-side feedback record |
| `Blocked private` | Feedback depends on private facts, assets, account state, or adapter details | keep local only |

## Non-Claims

- No automatic Core patch is created.
- No external task is written.
- No adapter is connected.
- No private Workspace fact is sent to Core.
