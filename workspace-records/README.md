# Workspace Records

Generated by: `scripts/utils/create_workspace_deployment.py`

This folder stores Workspace evidence for Core adoption, implementation, decisions, checks, and feedback.

| Folder | Purpose |
|---|---|
| `upgrades/` | Core release adoption records |
| `implementation/` | Technical landing notes |
| `decisions/` | Workspace-local decisions |
| `checks/` | Verification snapshots |
| `capability-loads/` | Records of installed Core runtime capability files |
| `feedback/` | Sanitized feedback to WorkSeed Core |

Rules:

- Keep project facts inside this Workspace.
- Keep reusable Core gaps sanitized before returning them to WorkSeed Core.
- Self-repair locally before sending anything to Core.
- Send Core only reusable gaps, stripped of private Workspace facts.
- Do not copy from legacy or historical Workspaces.
- Do not claim product-grade, source-equivalent, or real adapter connection from this fresh package.
- Do not claim `@prism`, `@mark`, or `@mira` runtime availability until
  `.codex/agents/**` and `.agents/skills/**` are installed or verified.

## Progressive Detail Capture

When a missing detail becomes useful, ask one natural-language question, then write the answer into the smallest correct record:

- identity and base facts -> `workspace-system-manifest.md`
- first project goal -> `projects/starter/plan.md`
- local decision -> `workspace-records/decisions/`
- Core capability runtime load -> `workspace-records/capability-loads/`
- reusable Core gap -> `workspace-records/feedback/`

## Built-In Feedback Interface

Use this folder for three related jobs:

1. Self-repair notes: what broke, how the Workspace fixed it locally, and what evidence proves it.
2. Local judgment: why the issue stays Workspace-local or should become a Core signal.
3. Sanitized Core signal: reusable issue summary with private facts removed.

Default classification:

| Classification | Meaning | Core action |
|---|---|---|
| `Workspace self-repair` | The issue belongs to local facts, local workflow, or local material | none |
| `Workspace record only` | The issue is worth remembering locally but is not reusable | none |
| `Core sanitized signal` | The issue may affect other Workspaces or the Template | review later |
| `Blocked private` | The issue cannot be shared without private context | none |
