## Start Here

1. Read `WORKSPACE.md`.
2. Read `workspace-system-manifest.md`.
3. Read `workspace-records/README.md`.
4. Read `docs/workspaces/initialization-and-core-loading.md`.
5. Read `docs/workspaces/network-health-and-reconnect-guide.md` if the user mentions reconnects, network instability, ports, proxy, VPN, or `.codex/.env`.
6. Classify the session as greeting/initialization, network health, local use, Core adoption, capability runtime loading, local adaptation, feedback to Core, or external adapter work.
7. Write durable evidence to the smallest correct Workspace surface.
8. Use the built-in feedback interface before sending anything back to Core.

## Workspace Boundary

Do not copy private project materials, raw assets, account state, private links, real adapter configuration, or external task-system contents into WorkSeed Core.

Do not overwrite `projects/**`, `WORKSPACE.md`, `workspace-system-manifest.md`, or `workspace-records/**` without a Workspace-owned reason and verification.

Adapter state is `Mock/read-only only` until a separate adapter stage is authorized and verified. External task writes are `Not required`.

## Network Health And Reconnects

If the user reports repeated reconnects, unstable network, proxy trouble, port
conflict, or asks about `.codex/.env`, stop expanding project work in the main
thread. Keep this thread as the source/control thread and ask the user to open
a narrow network health guidance thread with
`templates/network-health-guidance-thread-prompt.md`.

The known successful route is Workspace-local `.codex/.env` port or proxy
control followed by Codex restart/reopen. Do not commit a real `.codex/.env`.
Use `templates/codex-env-network-template.txt` as the copy source and keep
secret values redacted in reports.

Network health guidance is a local diagnostic workflow. It does not install
Core, modify project facts, connect adapters, write external task systems, or
prove a permanent fix until the user confirms reconnects stopped.

## Built-In Feedback Interface

When something breaks or feels incomplete:

1. Try local self-repair inside the Workspace first.
2. Record the local decision under `workspace-records/decisions/`.
3. Record repair evidence under `workspace-records/feedback/` when useful.
4. Send a sanitized signal to Core only when the issue is reusable beyond this Workspace.
