Generated by: `scripts/utils/create_workspace_deployment.py`

## Start Here

1. Read `WORKSPACE.md`.
2. Read `workspace-system-manifest.md`.
3. Read `workspace-records/README.md`.
4. Read `docs/workspaces/initialization-and-core-loading.md`.
5. Classify the session as greeting/initialization, local use, Core adoption, capability runtime loading, local adaptation, feedback to Core, or external adapter work.
6. Write durable evidence to the smallest correct Workspace surface.
7. Use the built-in feedback interface before sending anything back to Core.

## Workspace Boundary

Do not copy private project materials, raw assets, account state, private links, real adapter configuration, or external task-system contents into WorkSeed Core.

Do not overwrite `projects/**`, `WORKSPACE.md`, `workspace-system-manifest.md`, or `workspace-records/**` without a Workspace-owned reason and verification.

Adapter state is `Mock/read-only only` until a separate adapter stage is authorized and verified. External task writes are `Not required`.

## Built-In Feedback Interface

When something breaks or feels incomplete:

1. Try local self-repair inside the Workspace first.
2. Record the local decision under `workspace-records/decisions/`.
3. Record repair evidence under `workspace-records/feedback/` when useful.
4. Send a sanitized signal to Core only when the issue is reusable beyond this Workspace.
